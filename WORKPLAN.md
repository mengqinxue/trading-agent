# Trading-Agent 工作计划

> 基于 Karpathy Skills 原则，使用 Claude Code 分步执行

## 项目规范

### 代码规范
- 每个节点函数必须有开始/结束日志
- 使用 `trading_agent.utils.logger` 统一日志
- 遵循现有代码风格（不做无关修改）
- 类型注解完整

### 文档规范
- 每步完成后更新相关文档
- 文档放在 `docs/` 目录
- 使用中文注释和文档（本项目面向中文用户）

### Commit 规范
- 每完成一个任务就 commit
- Commit message 格式：`feat(node): 实现 screener 业务逻辑`
- 不做大批量 commit

### 测试规范
- 每个节点实现后必须写测试
- 测试放在 `tests/` 目录
- 使用 pytest

## 执行流程

每个任务执行时：

1. **我（富贵）准备**：
   - 明确任务范围
   - 定义验收标准
   - 准备必要上下文

2. **调用 Claude Code**：
   - 传递具体任务
   - 限定修改范围
   - 要求输出可验证结果

3. **验收**：
   - 运行测试
   - 检查日志
   - 检查代码规范

4. **推进**：
   - Commit
   - 更新文档
   - 更新 TODO状态
   - 进入下一任务
