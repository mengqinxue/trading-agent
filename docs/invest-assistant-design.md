## 问题陈述

**核心问题**：散户做波段/中频股票投资时，面临信息过载、决策碎片化、情绪干扰三大痛点。

- 信息来源分散：新闻、行情、公告、社交媒体，每天数百条，筛选成本高
- 决策缺乏系统：买入/卖出往往是直觉或跟风，缺乏结构化分析流程
- 情绪干扰严重：贪婪/恐惧主导，容易追涨杀跌，错过真正的机会

**目标用户**：
- 有一定投资经验的个人投资者（非小白）
- 持仓规模 10万-100万，关注波段机会而非高频套利
- 时间有限（上班族），希望高效获取决策建议

---

## 需求证据

**用户反馈（孟钦学）**：
- 已有持仓管理系统（stock/），有股票数据库
- 已有新闻聚合系统（TrendRadar/），但两者未联动
- 想要：粗筛 → 分析 → 博弈 → 决策的完整流程
- 想要：两个 agent 辩论 100 轮，评委拍板

**市场观察**：
- 股票投资 AI 工具市场存在，但多为单一功能（行情分析、新闻推送）
- 缺乏"多视角博弈 + 决策评审"的完整决策闭环

---

## 产品定位

**一句话**：一个帮散户做波段投资决策的 AI 系统，通过多 agent 博弈机制，让决策更理性、更系统。

**核心价值**：
- 信息过滤：从海量新闻中筛选真正相关的
- 多视角分析：技术面 + 基本面双轨并行
- 博弈决策：买卖双方辩论，评委拍板
- 持仓联动：结合已有持仓状态给出建议

---

## 核心功能模块

### 1. 热点发现（粗筛）

**输入**：全网热点新闻（TrendRadar MCP）
**输出**：候选股票池（20-50只）

**功能**：
- 获取 11 平台热搜 + RSS 订阅
- AI 分析热点板块（哪些板块在炒？）
- 识别炒作龙头（不是基本面龙头）
- 结合用户持仓关键词过滤

### 2. 数据分析 Agent（技术面）

**输入**：候选股票池 + 行情数据（akshare）
**输出**：买入/卖出/观望 + 置信度 + 论据

**功能**：
- K线形态/趋势识别
- 成交量/资金流分析
- 均线/支撑阻力位
- MACD/RSI 等技术指标
- 板块联动分析

### 3. 尽调分析 Agent（基本面）

**输入**：候选股票池 + 公司数据 + 新闻
**输出**：买入/卖出/观望 + 置信度 + 论据

**功能**：
- 公司基本面（财务、业务）
- 行业地位/竞争格局
- 政策/消息影响评估
- 龙虎榜/机构持仓
- 风险提示

### 4. 博弈辩论引擎

**输入**：两个 agent 的分析结果
**输出**：辩论摘要 + 双方核心论点

**机制**：
- 买入方提出论据 → 卖出方反驳
- 每轮交换观点、引用新数据
- 逐步收敛到共识或保留分歧
- 100 轮辩论（可配置）

### 5. 评委决策

**输入**：辩论结果 + 用户持仓状态
**输出**：最终决策 + 仓位建议

**功能**：
- 综合双方论据
- 结合持仓状态（已持有/空仓/仓位比例）
- 评估风险收益比
- 给出：买入/卖出/观望
- 给出仓位建议（如：买入 20% 仓位）
- 输出结构化决策报告

## 用户交互

### 每日运行流程

```
盘前 (9:00) → 自动运行粗筛 + 分析
盘后 (15:30) → 自动运行辩论 + 评审
推送 → 飞书/邮件推送决策报告
```

### Dashboard

- 查看候选股票池
- 查看每个股票的分析过程（辩论记录）
- 查看决策历史
- 管理持仓配置

## MVP 策略

**Phase 1（最小可用）**：
- 粗筛 agent（TrendRadar 集成）
- 单一分析 agent（技术面）
- 简单决策输出

**Phase 2**：
- 双轨分析（技术面 + 基本面）
- 博弈辩论引擎

**Phase 3**：
- 评委决策
- Dashboard + 飞书推送
- 定时运行
